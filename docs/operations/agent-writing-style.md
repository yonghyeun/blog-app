# Agent Writing Style

Repo-local Markdown 문체 기준.

## Purpose

에이전트가 작성하는 운영 문서, issue handoff, closeout receipt, skill 문서의
읽기 부담을 줄인다.

## Default Style

- 한국어 우선.
- 짧은 문장 우선.
- 불렛 중심.
- 한 불렛에 하나의 판단만 배치.
- 긴 배경 설명보다 현재 상태와 다음 행동 우선.
- 명사형 또는 단문형 종결 선호.
- 기술 고유명사, 명령어, 파일 경로, API 이름은 English token 유지.

## Preferred Endings

- `완료.`
- `필요.`
- `가능.`
- `리스크 존재.`
- `추가 확인 필요.`
- `근거 부족.`
- `범위 아님.`
- `후속 issue 필요.`

## Avoid

- `~하였습니다`
- `~할 수 있습니다`
- `~하는 것이 좋습니다`
- `~라고 볼 수 있습니다`
- 긴 완곡어.
- 같은 의미를 반복하는 배경 설명.
- 판단과 행동이 섞인 긴 문장.

## Structure

필요한 축만 사용.

- 상태: 지금 무엇이 사실인지.
- 근거: 왜 그렇게 판단했는지.
- 결정: 무엇을 선택했는지.
- 다음 행동: 무엇을 해야 하는지.
- 비범위: 이번 작업에 넣지 않을 것.

## Examples

Before:

```text
해당 작업은 기존 문서 작성 방식이 일관되지 않았기 때문에 새로운 문서 작성
스킬을 추가하는 것이 좋습니다.
```

After:

```text
- 현재 문제: 문서 작성 방식 불일치.
- 결정: `korean-doc-writer` skill 추가.
- 목적: 읽기 부담 감소.
```

Before:

```text
이 부분은 추후에 추가적으로 확인할 수 있습니다.
```

After:

```text
추가 확인 필요.
```

## Exceptions

- 법적 문서, 고객-facing 안내, 외부 공개문처럼 정중한 평문이 필요한 경우.
- 문학적 글, 에세이, 제품 카피처럼 의도적으로 리듬이 필요한 경우.
- 기존 문서의 톤을 보존해야 하는 좁은 수정.

예외 사용 시 source issue 또는 handoff에 이유 기록.
